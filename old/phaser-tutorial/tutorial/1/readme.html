1er Paso: Armando el esqueleto

El HTML ./{{version.directory}}/index.html

Phaser es una librería para hacer videojuegos para jugar en el navegador. Como el juego que harás aparecerá en el navegador, necesitamos tener un "recipiente" en dónde poner el juego. Este recipiente es una página html.

Pero primero necesitarás descargar phaser! http://phaser.io/download/stable (solo tienes que hacer click en el botón que dice "min.js").

Esta librería phaser.min.js, debes cargarla en tu página web, antes del script que contiene tu juego.

Dentro del body del html, debes incluir un div con un id. No olvides el nombre del id.

El script con el código de tu juego ./{{version.directory}}/game.js

Ya tienes un recipiente! ahora necesitas un archivo .js con el código que define la lógica de tu juego. Ese archivo también tiene una estructura inicial importante.

Primero, necesitarás indicarle a phaser, en cuál div debe "dibujar" el juego. Esto se hace mediante new Phaser.game(...). game es el que dibuja el juego, y se encarga de ejecutar toda la lógica.

Al crear el juego, le estamos enviando 4 parámetros: el primero indica el ancho del juego, el segundo indica el alto del juego, y el último, el id del div en el que phaser debe dibujar el juego.

Phaser se maneja mediante niveles. Y es recomendable que todo nivel tenga, al menos, estos tres métodos: preload, create y update.

Pero, por hora, la variable level1, solo es una variable cualquiera, porque no le hemos indicado a game, que level1 es un nivel, ni tampoco le hemos dicho cuándo debería comenzar.

Lo que hace cada uno de estos métodos, y cómo agregar el nivel al juego, lo veremos en la siguiente versión ;)
